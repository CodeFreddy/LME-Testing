# LME Matching Rules - BDD Test Suite (Prototype)

## Overview

This is a prototype demonstrating the **Maker → Checker → BDD → Scripts** pipeline for LME Matching Rules testing.

## Generated Structure

```
lme-testing/
├── features/
│   ├── support/
│   │   ├── env.rb          # Cucumber setup, LME client initialization
│   │   └── hooks.rb        # Test lifecycle hooks
│   ├── step_definitions/
│   │   └── matching_rules_steps.rb  # Generated step definitions
│   └── matching_rules/
│       └── SR-MR-003-01_price_validation_failure.feature  # Generated scenarios
├── lib/
│   ├── lme_client.rb       # LME API client (prototype)
│   ├── lme_api.rb           # LME REST API (prototype)
│   └── lme_post_trade.rb    # Post-trade operations (prototype)
├── reports/                  # Test output reports
├── screenshots/              # Failure screenshots
├── Gemfile
├── cucumber.yml
└── Rakefile
```

## Quick Start

### Setup

```bash
# Install dependencies
bundle install

# Set environment variables (optional)
export LME_API_URL=https://lme-test.internal/api/v1
export LME_USERNAME=test_user
export LME_PASSWORD=test_password
export HEADLESS=true  # Run browser in headless mode
```

### Run Tests

```bash
# Run all tests
bundle exec rake test:all
bundle exec cucumber

# Run with specific profile
bundle exec cucumber --profile api
bundle exec cucumber --profile web
bundle exec cucumber --profile smoke

# Run specific feature or scenario
bundle exec cucumber features/matching_rules/SR-MR-003-01.feature
bundle exec cucumber features/matching_rules/SR-MR-003-01.feature:7  # line 7

# Run with tags
bundle exec cucumber --tags '@post_trade'
bundle exec cucumber --tags '@smoke and not @skip_regression'
```

## Pipeline Integration

This test suite is generated by the LME-Testing pipeline:

```
semantic_rules.json
    ↓ (maker)
maker_cases.jsonl
    ↓ (checker)
checker_reviews.jsonl + coverage_report.json
    ↓ (bdd)
bdd_cases.jsonl + features/*.feature + *_steps.rb
    ↓ (script generation - future)
runnable Cucumber tests with real API clients
```

## Generated Files

### Feature Files (*.feature)
Gherkin scenarios with:
- Scenario IDs for traceability
- Tags for test categorization
- Evidence comments linking to atomic rules

### Step Definitions (*_steps.rb)
Ruby step definitions with:
- Real LME API calls (prototype implementations)
- RSpec expectations
- Proper error messages

## Tags

| Tag | Purpose |
|-----|---------|
| `@api` | Tests using API client only |
| `@web` | Tests requiring browser (Watir/Selenium) |
| `@auth` | Tests requiring authentication |
| `@post_trade` | Post-trade operation tests |
| `@smoke` | Fast smoke tests |
| `@regression` | Full regression tests |
| `@positive` | Happy path tests |
| `@negative` | Negative scenario tests |
| `@boundary` | Boundary condition tests |
