Field Guide

  • Decision: final human action. If it is not pending, the system executes this action.
  • Block Recommendation Review: human review of the checker blocking recommendation. It is used for audit only and does not override Decision.
  • Issue Types: open the panel and select one or more checkboxes. These tags are used for stats and targeted rewrite guidance.
If Decision is already set to approve, rewrite, or reject, the system follows that decision even if Block Recommendation Review is still pending_review.

Issue Type Reference

  • rule_misread: 规则理解错误 - 测试案例误解了 semantic rule 的核心含义或适用条件。
  • bad_assertion: 断言不合理 - Then 断言不准确、不充分，或无法证明规则是否满足。
  • weak_evidence: 证据不足 - 引用的 evidence 过弱、过长或没有逐条对应到当前场景。
  • wrong_case_type: 案例类型错误 - case_type 与规则要求不匹配，例如把边界案例写成正例。
  • unclear_steps: 步骤不清晰 - Given/When/Then 表达模糊，阅读者难以理解或执行。
  • missing_negative_case: 缺少负例 - 当前规则需要 negative case，但 maker 没有提供有效负例。
  • missing_boundary_case: 缺少边界案例 - 当前规则需要 boundary case，但 maker 没有提供有效边界测试。
  • non_executable_scenario: 场景不可执行 - 场景无法在测试平台中落地执行或无法验证结果。
  • coverage_dispute: 覆盖判断争议 - checker 对 covered/partial/uncovered 的判断与人工理解不一致。
  • duplicate_coverage: 重复覆盖 - 多个案例重复覆盖同一 slot，但遗漏了其他必选 case type。
