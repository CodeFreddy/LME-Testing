# 1. 代码场景（最省额度）
```
请一次性完整给出可直接运行的代码，不要分步、不要省略、不要提问确认。
需求：【这里写你的功能】
技术栈：【Node.js/Java/Python等】
要求：
- 完整代码，无省略
- 带必要注释
- 直接可用，不需要我补充
- 若有依赖，列出安装命令
- 若有潜在问题，一并说明
```

# 2. 改 Bug / 查问题
```
下面是代码与报错，你直接给出**完整修复后的代码**，不要提问、不要分步。
代码：
【粘贴代码】

报错：
【粘贴报错】

要求：一次性给出最终可用版本，解释改动原因。
```

# 3. 项目结构 + 代码一次性生成
```
帮我设计一个【项目名称】的完整结构，并一次性给出所有关键文件代码，不要分步骤、不要省略文件。
功能：【写清楚】
技术栈：【】
输出格式：每个文件路径+完整代码。
```

# 4. 总结 / 文档 / 分析
```
请一次性完整总结，不要分段、不要反问、不要让我补充信息。
内容：【粘贴内容】
要求：
- 结构清晰
- 关键点提炼
- 结论直接给出
- 不追加问题
```

# 5. 上传文件后（PDF/代码包）
```
我已上传文件，请你**一次性完整分析**，不要分轮、不要提问、不要等待我补充。请直接给出：
1. 核心内容总结
2. 关键要点
3. 可直接使用的结论/代码/建议
```

# 省额度核心技巧（非常重要）
1. **一次把话说完**，别拆多轮
2. 强制要求 **“一次性给完整结果”**，Claude 就不会反问你
3. 长内容尽量**一次贴完**，不要分好几次发
4. 一个会话里尽量**只聊一个主题**，别频繁开新话题
5. 触达额度后**等5小时再用**，别狂刷，越刷越紧
