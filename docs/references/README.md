# Reference Materials

**Status:** Active references index
**Date:** 2026-05-08
**Canonical owner:** `docs/index.md` and `docs/governance/REFERENCE_POLICY.md`
**Scope:** Historical, external, chat-derived, and exploratory reference materials.

## Purpose

This directory stores reference materials that informed or influenced the repository documentation and planning.

These files are kept for:

- historical context,
- external comparison,
- traceability of ideas,
- and future review.

They are **not** the source of truth for current repository governance.

For current policy and implementation guidance, use the official docs in `docs/`.

---

## How to Use This Directory

Use reference materials to:

- understand background thinking,
- compare older or external proposals,
- recover rationale for certain decisions,
- identify ideas that may later be promoted into official docs.

Do **not** use these files as a substitute for:
- `docs/planning/roadmap.md`
- `docs/planning/implementation_plan.md`
- `docs/architecture/architecture.md`
- `docs/governance/acceptance.md`
- `docs/governance/model_governance.md`
- `docs/governance/agent_guidelines.md`
- `docs/governance/testing_governance.md`
- `docs/governance/prompt_lifecycle.md`
- `docs/planning/step_integration_plan.md`

## Governance Rule

If a reference file conflicts with the official docs, the official docs win.

If a reference file contains useful ideas, those ideas should be incorporated into the official docs explicitly rather than left as hidden policy inside a raw reference file.
